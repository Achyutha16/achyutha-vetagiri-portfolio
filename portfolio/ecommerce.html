Project

eCommerce Shipping Software

It is cloud-based eCommerce shipping software that offers advanced reporting and analytics to help you streamline eCommerce operations across sales channels. It includes discounted carrier rates, shipping presets, automation rules, barcode scanning, and no seller-fulfilled Prime fees with its base plan. Higher-level plans include customized workflows and integrations, vendor portals, supplier management, dedicated customer service reps, and more.

My work

As a UI designer for eCommerce Shipping Software, I designed the user interface screens based on the requirements gathered from stakeholders.

Here are some of the key screens that I designed:

  • Dashboard Screen: This screen should display key metrics such as order volume, revenue, and shipping performance. It should also provide quick access to frequently used features and reports.
  • Order Management Screen: This screen should allow users to easily view, filter, and manage orders across sales channels. It should also enable users to create and print shipping labels and track packages.
  • Reporting and Analytics Screen: This screen should provide detailed insights into sales and shipping performance, including data such as revenue by sales channel, shipping costs, and fulfillment times. Users should be able to customize reports and generate data visualizations to help them make informed decisions.
  • Settings and Preferences Screen: This screen should allow users to customize their experience and set up their shipping preferences, such as carrier selection and package dimensions.
  • Help and Support Screen: This screen should provide users with access to documentation, FAQs, and a support portal.

User Research

When conducting UX research on an eCommerce Shipping Software, there are several key points of concern that should be considered:

Mobile optimization: Many users access recruitment tools on mobile devices

Competitor analysis: what other recruitment tools are available in the market

Technical constraints:limited functionality or need to integrate with existing systems

Balance competing interests: while still prioritizing the needs of the users

Create user journey maps:to improve the user experience.
